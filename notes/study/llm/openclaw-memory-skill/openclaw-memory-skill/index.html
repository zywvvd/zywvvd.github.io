本文记录如何配置 OpenClaw 的记忆系统，以及创建、打包和测试自定义技能的完整流程。

简介

OpenClaw 是一个可扩展的 AI 助手框架，核心特性包括：

  • 记忆系统：持久化存储对话中的重要信息，支持语义搜索
  • 技能系统：将特定工作流程打包成可复用的模块
  • 多 Agent 支持：可配置多个独立运行的 AI 助手

本文将详细介绍：

  1. 记忆系统的工作原理与配置方法
  2. 本地 Embedding 的安装与配置
  3. 技能的创建、打包与更新流程
  4. 实战示例与最佳实践

记忆系统详解

为什么需要记忆系统？

AI 助手默认只记得当前对话的内容，一旦对话结束或上下文被压缩，之前的信息就会丢失。记忆系统解决了这个问题：

场景 没有记忆 有记忆
用户偏好 每次都要重新说明 自动记住并应用
项目配置 每次都要重复 自动识别并使用
历史决策 无法追溯 可以随时查询
跨对话协作 不可能 可以引用之前的内容

记忆文件结构

OpenClaw 的记忆是纯 Markdown 文件，存放在 Agent 的工作空间中：

MEMORY.md vs memory/ 目录：

文件 用途 读取时机
MEMORY.md 核心长期记忆，重要信息 会话开始时
memory/YYYY-MM-DD.md 每日日志，临时记录 读取今天和昨天
memory/topic.md 主题记忆，特定领域 通过语义搜索调用

记忆工具

OpenClaw 提供两个记忆工具：

1. memory_search - 语义搜索

返回相关记忆片段，包含：

  • path - 文件路径
  • startLine / endLine - 行号范围
  • snippet - 内容片段
  • score - 相关度分数

2. memory_get - 读取文件

直接读取指定记忆文件的内容。

语义搜索配置

默认行为 vs 语义搜索

默认情况下，memory_search 使用全文匹配（FTS），只能通过关键词查找。这意味着：

  • 搜 “博客” 找不到 “Hexo”
  • 搜 “发布” 找不到 “deploy”
  • 搜 “图片” 找不到 “image”

语义搜索通过 embedding 向量化，可以理解语义相似性，大幅提升搜索准确度。

Embedding Provider 选择

OpenClaw 支持多种 embedding provider：

Provider 费用 速度 准确度 推荐场景
local 免费 中等 良好 个人使用、隐私优先
openai 付费 快 优秀 商业使用、高准确度需求
gemini 付费 快 优秀 已有 Gemini API
ollama 免费 慢 良好 本地 Ollama 环境

3. 添加资源文件

scripts/ 目录：

  • 放置可执行脚本
  • 使用 #!/bin/bash 或 #!/usr/bin/env python3 开头
  • 添加执行权限：chmod +x scripts/*.sh

references/ 目录：

  • 放置详细文档
  • 被 SKILL.md 引用时按需加载
  • 避免与 SKILL.md 内容重复

assets/ 目录：

  • 放置模板文件
  • 用于生成最终输出

最佳实践

记忆系统

  1. 定期整理 MEMORY.md

    • 移除过时信息
    • 更新项目路径
    • 补充新的偏好

  2. 使用语义化标签

  3. 及时写入重要信息

技能系统

  1. description 要具体

    • 说明技能做什么
    • 列出触发关键词
    • 避免模糊描述

  2. 保持 SKILL.md 精简

    • 核心流程放 body
    • 详细信息放 references
    • 避免内容重复

  3. 脚本要健壮

    • 添加错误处理
    • 提供使用说明
    • 记录关键日志

  4. 版本控制

    • 使用 git 管理技能
    • 保留更新历史
    • 便于回滚
